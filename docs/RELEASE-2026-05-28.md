# Release Note — 報價單 / 合約 / 收款管理強化

**版本期間：** 2026-05-26 ~ 2026-05-28
**範圍：** 後台 CRM（報價單、合約、發票收款、系統設定）

## 重點摘要

- 合約／範本內容改回**純文字並正確分段**，新增**佔位符自動替換**（客戶名、金額、日期等）
- 報價單「快速建立」修正按鈕顯示與選取 bug；合約**新增同款快速建立面板**
- 報價單轉合約**自動套用合約範本正文**；報價/合約共用程式碼抽成 partial
- 合約**簽署流程強化**：狀態守衛、送出（標記/Email）、上傳客戶回簽檔
- 收款改為**逐筆明細帳本**（合約與發票共用），可登記/檢視/刪除
- 報價單**標準條款集中於後台管理**，不再寫死

---

## 1. 合約與範本內容：分段排版 + 佔位符

- 合約與合約範本的「內容」改回**純文字編輯**（移除會壓掉換行的 TinyMCE），詳情頁與 PDF 以 `white-space: pre-line` 正確呈現段落（已驗證 dompdf 支援）。
- 合約建立/更新時，正文中的 `{{client_name}}`、`{{amount}}`、`{{start_date}}` 等**佔位符自動替換為實際值**；無對應來源的佔位符（如 `{{yearly_fee}}`）原樣保留供人工填寫。
- *Commits:* `48d3227`, `b355922`

## 2. 報價單快速建立修復

- 修正方案按鈕**沒有名稱**（誤用不存在的 `name` 欄位）→ 改用 `title`。
- 修正選取方案時**同時亮起兩顆**（推薦方案固定實心底色）→ 改為一次只亮一顆，推薦改用徽章標示。
- *Commit:* `f477dbc`

## 3. 報價單 ↔ 合約 強化

- **報價單轉合約**：依合約類型自動套用合約範本正文並替換佔位符（取代原本的罐頭句）。
- **合約新增「從服務方案快速建立」面板**（與報價單一致，可一鍵帶入項目）；網站主方案改為**非必選**，方案或附加項目至少擇一即可。
- 報價/合約 create/edit 四個表單重複的「項目列 JS」與「快速建立面板」**抽成共用 partial**（淨減約 200 行）。
- *Commits:* `79d003d`, `36c89f7`

## 4. 合約簽署流程與收款機制

**簽署 / 動作**

- 狀態轉換**守衛**：不可任意跳轉，只走合法路徑。
- 送出給客戶兩種方式：**「標記為已送出」** 與 **「Email 寄送 PDF」**。
- **上傳客戶回簽合約**（PDF／掃描圖）→ 自動標記為「已簽署」並留存檔案，可於詳情頁檢視。

**收款**

- 新增**多型收款帳本**（`payments` 表），合約與發票**共用**；`已收金額` 一律由帳本加總同步。
- 合約詳情頁新增**「登記收款」**（金額/方式/日期/備註）與**收款明細**，可刪除並自動重算。
- 發票收款同步改走帳本（既有已收金額以資料遷移轉為「期初」帳本列，不遺失）。
- *Commit:* `e323cb5`

## 5. 標準條款集中管理

- 報價單「帶入標準條款」內容與 PDF 固定備註，改為後台 **「系統設定 → 單據條款」** 可編輯（不再寫死於程式）。
- *Commit:* `71b482a`

---

## ⚙️ 部署注意事項

1. **資料庫遷移**：`php artisan migrate`
   - 新增 `payments` 表、`contracts` 簽署檔欄位、舊 `paid_amount` 回填遷移
2. **種入預設設定**：`php artisan db:seed --class=SettingSeeder --force`（單據條款預設值）
3. **清快取**：`php artisan optimize:clear`
4. **Email 寄送合約**：需郵件設定正常，且 `ContractToClient` 為佇列寄送 → 需 queue worker 執行（或 `QUEUE_CONNECTION=sync`）
5. 部署路由亦可：`/deploy/migrate-seed`、`/deploy/optimize`

## ✅ 驗證狀態

- 後端邏輯（收款累加/付清/刪除、狀態守衛、發票回歸、佔位符替換、dompdf 分段）皆以腳本驗證通過
- 語法 / Pint / Blade 編譯 / 路由 / PHPStan（無新增實質錯誤）通過
- **建議上線前手動 UI 煙霧測試**：合約送出→上傳回簽→登記收款→查明細；報價/合約快速建立與項目增刪
