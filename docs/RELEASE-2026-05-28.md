# Release Note — 報價單 / 合約 / 收款管理大幅強化

**版本期間：** 2026-05-26 ~ 2026-05-28
**範圍：** 後台 CRM（報價單、合約、合約範本、發票收款、系統設定）
**Commit 數：** 21（最新 `2aeab3b`）

> 本版針對報價單與合約模組做了一輪完整稽核與強化：修正多項顯示與資料 bug、補齊缺漏的流程動作、統一報價↔合約↔發票的轉換與狀態邏輯、新增收款帳本與簽署流程，並把寫死的條款集中到後台管理。

---

## 重點摘要

- 合約／範本內容改回**純文字並正確分段**；條款佔位符改為**即時連動**（改項目/起迄自動反映）
- 合約**月/年項目依合約期間自動計算數量**
- 報價單**快速建立**修正按鈕顯示與選取 bug；合約**新增同款快速建立面板**
- 報價單轉合約**自動套用合約範本正文**；報價/合約共用程式碼抽成 partial
- 合約**簽署流程**：狀態守衛、送出（標記/Email）、上傳客戶回簽檔（紙本可免附檔）
- 收款改為**逐筆明細帳本**（合約與發票共用），可登記/檢視/刪除
- 報價單／發票**加上稽核軌跡**；報價單新增**快速狀態變更 + 過期排程**
- 標準條款、PDF 備註**集中於後台管理**；PDF 移除不該對客戶顯示的內部資訊
- 修正複製、刪除、轉換等流程的資料 bug，並加上交易保護

---

## 1. 合約與範本內容：分段排版 + 佔位符（即時連動）
- 合約與合約範本內容改回**純文字編輯**（移除會壓掉換行的 TinyMCE），詳情頁與 PDF 以 `white-space: pre-line` 正確呈現段落（已驗證 dompdf 支援）。
- 合約正文**保留 `{{client_name}}`、`{{amount}}`、`{{start_date}}` 等佔位符**，於詳情頁／PDF 顯示時才**即時帶入目前值**（`Contract::renderedContent()`）。**改項目價格或合約起迄後，條款自動反映最新金額／日期，不需重存**。
- **月/年項目依合約期間自動計算數量**：合約起迄變更、或項目單位設為「月／年」時，數量依期間自動帶入（報價單無起迄欄位故自動略過）。
- **佔位符資料來源補齊**：新增合約 `yearly_fee`「年度費用」欄位作為 `{{yearly_fee}}` 來源（純參考、不計入總額／收款）；`{{revision_limit}}` 改以固定文字呈現。至此範本無殘留未填佔位符。
- *Commits:* `48d3227`, `b355922`, `b2ab096`, `0c0a9dc`, `baaec19`

## 2. 報價單快速建立
- 修正方案按鈕**沒有名稱**（誤用不存在的 `name`）→ 改用 `title`。
- 修正選方案時**同時亮起兩顆**（推薦方案固定實心底色）→ 一次只亮一顆，推薦改用徽章。
- 網站主方案改為**非必選**（方案或附加項目至少擇一即可）。
- *Commits:* `f477dbc`, `36c89f7`

## 3. 報價單 ↔ 合約 ↔ 發票 整合
- **轉合約**：依合約類型自動套用合約範本正文＋佔位符替換（取代罐頭句）。
- **合約新增「從服務方案快速建立」面板**（與報價單一致）。
- 報價/合約 create/edit 四個表單重複的「項目列 JS」「快速建立面板」**抽成共用 partial**。
- **轉換一致性**：轉發票／轉合約都需報價單為「已接受」並標記 `accepted_at`；列表頁同時提供轉發票與轉合約。
- **轉合約可選範本**：詳情頁「轉為合約」以彈窗提供合約範本下拉，依所選範本決定合約類型與正文（列表快速轉換維持預設服務範本）。
- *Commits:* `79d003d`, `36c89f7`, `f8ff4fe`, `6dcb389`

## 4. 合約簽署與動作流程
- 狀態轉換**守衛**：不可任意跳轉。
- 送出客戶：**「標記為已送出」** 與 **「Email 寄送 PDF」** 兩種。
- **上傳客戶回簽檔**（PDF／掃描圖）→ 標記為已簽署並留存；**紙本可不附檔直接標記**。
- *Commits:* `e323cb5`, `84612fe`

## 5. 付款／收款機制
- 多型 **`payments` 收款帳本**（`Payment` + `HasPayments` trait），合約與發票**共用**；`paid_amount` 一律由帳本加總同步。
- 合約與發票詳情頁皆有「登記收款」（金額／方式／日期／備註／**憑證**）+ **收款明細表** + 刪除。
- 可上傳轉帳截圖／收據作為**收款憑證**（`proof_path`），明細提供檢視連結；刪除收款或主檔時連帶刪除憑證檔。
- 既有 `paid_amount` 由資料遷移轉為「期初」帳本列，不遺失。
- *Commits:* `e323cb5`, `b3573fa`, `4960fab`

## 6. 報價單狀態與過期
- 報價單新增**快速狀態變更**（狀態守衛；接受/拒絕自動記錄 `accepted_at`/`rejected_at`）。
- 新增 **`quotes:mark-expired`** 每日排程（00:35）自動標記過期報價單。
- *Commit:* `f8ff4fe`

## 7. 稽核軌跡（Audit Log）
- **Quote／Invoice** 套用 Spatie LogsActivity，詳情頁新增「活動紀錄」表格（與合約一致）。收款造成的金額/狀態變更亦會入帳。
- *Commit:* `f88b576`

## 8. 條款集中管理與 PDF 呈現
- 報價單「標準條款」與「PDF 備註」改為後台 **「系統設定 → 單據條款」** 編輯。
- 維護合約／一頁式合約範本**補上保密約定條款**。
- 合約 PDF **移除內部狀態 badge**；報價單 PDF 移除誤植的「簽約日期」、8 段寫死條款，以及**雙方簽名區**（報價單非簽約文件）。
- 報價單詳情頁顯眼呈現總金額。
- *Commits:* `71b482a`, `a35ba25`, `78955b7`, `8e6f527`, `84612fe`, `2aeab3b`

## 9. Bug 修正與穩定性
- **複製合約**不再沿用原合約的客戶回簽檔、`quote_id` 與合約期間（B1、O2）。
- **刪除合約／發票**連帶清除收款帳本與簽署檔，避免孤兒資料（B2）。
- `store/update/兩個 convert` 包進 **DB transaction**，避免半套資料（Q4）；報價單列表預載關聯避免 N+1。
- sidebar「單據條款」連結以 `Route::has` 包覆，避免部署當下 route 快取落後造成整個後台 500。
- *Commits:* `f8ff4fe`, `c21d105`, `84612fe`

## ⚙️ 部署注意事項（重要）

1. **資料庫遷移**：`php artisan migrate`
   - payments 表（含收款憑證 proof_path）、contracts 簽署檔欄位、舊 paid_amount 回填、合約範本保密條款資料遷移
2. **種入預設設定**：`php artisan db:seed --class=SettingSeeder --force`（單據條款預設）
3. **重建快取（必做）**：`php artisan optimize:clear && php artisan optimize`
   - 本版新增多條路由（合約 send/email/sign/payments、報價單 status、設定 documents）；
     **route 快取未重建會造成詳情頁 500**。
4. **Email 寄送合約**：需郵件設定正常，且 `ContractToClient` 為佇列寄送 → 需 queue worker（或 `QUEUE_CONNECTION=sync`）。
5. **排程**：`quotes:mark-expired`、`invoices:mark-overdue` 需 Laravel scheduler cron 運行。
6. 一鍵部署：`/deploy/init?token=<DEPLOY_TOKEN>`（含 migrate + seed + optimize + view:cache）。
